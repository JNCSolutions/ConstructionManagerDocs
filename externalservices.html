Preparing your Network

It is important that your firewall and router are properly configured to let traffic from constructionmanager.net through to your MySQL database and Sage integration service.

Ports 3306 (for MySQL) and 8081 (for the Sage integration service) must be open on your router and forwarded to the correct machine. If that machine has an additional firewall such as Windows Firewall, rules must be set to allow connections to those ports or programs.

Connecting to Construction Manager 50

Construction Manager currently uses the data that you have stored in your CM50 database.

You (or your IT department) will need to supply Construction Manager with the MySQL credentials used by your CM50 software.

Hostname The IP address of the MySQL server.
Username The username associated with the CM50 account.
Password The password associated with the CM50 account.
Port The port that MySQL is listening on (usually 3306).
Database The database containing the company data.

Connecting to Sage

Construction Manager interacts with your sage data by means of a service listening on your network. When installed, this service will run on your server waiting for connections from https://constructionmanager.net

The Sage integration service can be configured with the provided settings utility. The settings utility allows you to configure the following fields:

Sage path The path to your sage data.
Sage username The username of your sage user.
Sage password The password of your sage user.
Service password This value needs to be set both on the server and the website. The only requirement is that they match, as this is the password used to allow the website to interact with your sage data.
Sage version The version of the sage SDO version to use.

The Sage integration service listens on port 8081, so it is important that you or your IT ensure that this port is open on your router/firewall. When you're comfortable that the service and router are correctly configured, you can fill in the Sage Integration Settings section of your connection settings page.

NB: If you have filled in your Sage connection details but they are incorrect, you will be prompted to fix it each time the system is accessed.

Troubleshooting

The service won't start on my machine

  • Check that the appropriate SDO is installed on your server.
  • Check the Windows Event Log to see whether the SDO TLB file must be registered.
